# PGAUTH-005 — External Source Authority Contract

Task: `PGAUTH-005` ("External source authority contract bounded") — parent `PGAUTH-001`, phase `2-postgres-foundation`.
Status: **owner-ratified target contract; implementation incomplete**. `STORE-001` revision 2 fixes the boundary between external issuers/physical sources and PostgreSQL-owned LifeOS product records. This document performs no external write, credential read, secret migration, or production cutover.

Scope: how external issuers, services, devices, repositories, and human approvals synchronize with LifeOS without unresolved dual-write ownership. The external source remains authoritative for facts it alone controls, while all durable LifeOS product bytes, protected encrypted secret bytes, custody metadata, derived records, and operational history belong in PostgreSQL + RuVector. Plaintext secrets remain outside every durable capture surface.

## Contract invariants (apply to every row)

1. **External fact authority and LifeOS durable ownership are distinct.** A device, issuer, human, or remote controls its external fact; PostgreSQL retains all durable bytes, metadata, custody, and operational records LifeOS depends on.
2. **Secret plaintext capture is absolutely prohibited; encrypted custody is mandatory.** Protected encrypted bytes belong in PostgreSQL, while plaintext exists only at the authorized lease-bound target and never in redb, logs, UI, model context, proof, Git, or `.rvf`.
3. **No unresolved dual-write.** Every edge names one canonical side and one declared replica/cache, with an explicit synchronization direction, conflict policy, and freshness rule.
4. **An approval gate + outage recovery is declared per family.** Human-approved records are immutable and re-solicited, not fabricated; service outages fail closed rather than fall back to a stale copy that could masquerade as authority.
5. **Unresolvable external evidence proves nothing.** A citation without a resolvable, checksummed source is flagged, never counted (EDGE-CLAIM-005, NUPG-CLAIM-010, ENVPROJ-CLAIM-007, all verified).
