# PGAUTH-005 — External Source Authority Contract (DRAFT)

Task: `PGAUTH-005` ("External source authority contract bounded") — parent `PGAUTH-001`, phase `2-postgres-foundation`.
Status: **draft, planning-only**. Every boundary below is a *proposed* contract and remains **pending STORE-001** (the David-approved store-relationship decision). No external system is connected, no credential is read, and no secret value is captured by this document.

Scope: how externally authoritative services, devices, repositories, credential stores, and human-approved records synchronize with LifeOS **without copying secret values and without creating unresolved dual-write ownership**, per the PGAUTH-005 goal. For every external family, the external system remains the authority; LifeOS holds a **local identity reference plus permitted cached metadata only**.

Evidence base: `generated/notebooklm_source_claims.source.csv` (claim map), `generated/notebooklm_claim_verification/NBVERIFY-001.truth-matrix.csv` + `NBVERIFY-003.truth-matrix.csv`, and — for the credential/secret family, which the NotebookLM claim map never covers — the **envctl source tree** (`crates/secretd`, `crates/secretctl`, `crates/secrets-engine`, `crates/secrets-store-libsql`) adopted via ADOPT-002. **Coverage gap flagged:** no `*-CLAIM-*` row in the map mentions secrets or credentials; that family's invariants are derived from the PGAUTH-005 gate and envctl primary source, not from a NotebookLM claim.

## Contract invariants (apply to every row)

1. **The external system is the authority; LifeOS holds a reference + permitted metadata only** (PGAUTH-005 gate). LifeOS never silently becomes the canonical owner of external truth.
2. **Prohibition on secret-value capture is absolute.** No credential, token, or key value enters any LifeOS store, planning artifact, projection, or `.rvf`. Only references (service+account) and handles resolved at point-of-use are permitted.
3. **No unresolved dual-write.** Every edge names one canonical side and one declared replica/cache, with an explicit synchronization direction, conflict policy, and freshness rule.
4. **An approval gate + outage recovery is declared per family.** Human-approved records are immutable and re-solicited, not fabricated; service outages fail closed rather than fall back to a stale copy that could masquerade as authority.
5. **Unresolvable external evidence proves nothing.** A citation without a resolvable, checksummed source is flagged, never counted (EDGE-CLAIM-005, NUPG-CLAIM-010, ENVPROJ-CLAIM-007, all verified).
